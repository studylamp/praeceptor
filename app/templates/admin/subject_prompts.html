What happens on each message

  1. The gate ({{ gate_model }}) checks the message is schoolwork for an enrolled subject. It only classifies — it never answers. If it can't produce a valid verdict, the tutor is withheld (it fails closed, never open).
  2. If the message is on subject, the tutor ({{ tutor_model }}{% if model_inherited %} — inherited from the app default{% else %} — pinned for this subject{% endif %}) answers. It receives its system prompt, the conversation so far, and the new message. Long chats are trimmed: once the history passes {{ max_history_kept }} turns it is cut back to the most recent {{ max_history_turns }} (so the model sees at most {{ max_history_kept }} turns, and normally at most roughly {{ "{:,}".format(max_history_tokens) }} estimated tokens) — older turns drop out of the model's view but stay in your conversation log. Messages the gate blocked are never re-sent to either model; they too exist only in your log.

  3. This subject's tutor can also run sandboxed Python to compute and verify answers; the tool definitions it sees are shown at the bottom.

(Two exceptions: after too many tool rounds in a single turn, a final wrap-up call runs without the tools so the turn always ends in a worded answer; and if a browser has JavaScript turned off, the app falls back to a simpler mode where the tutor answers without tools — that fallback also omits the "Computation tools" section of the system prompt above.)

{{ tool_specs_json }}

Fixed notes Praeceptor can add to a tool result

A tool's result goes back to the tutor as JSON: the code's own printed output (truncated if very long) or its error message (error messages have file paths scrubbed out first), to which the server can add the fixed notes below. If a tool call is malformed, times out, or the sandbox refuses it (for example, an attempt at network or file access), the result is instead a short plain-English notice saying what went wrong, so the tutor can correct itself; and when a solves check fails, verify also reports the solution set it actually found. For the verify tool, the checking code is written by the server, never by the tutor, so the tutor cannot grade its own work.
